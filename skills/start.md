---
name: om:start
description: 启动新的任务执行周期
---

<NO-OTHER-SKILLS>
执行此技能时，不得调用 superpowers、gsd 或其他任务编排相关的技能。OpenMatrix 独立运行，不依赖外部任务编排系统。
</NO-OTHER-SKILLS>

<objective>
解析任务文档，通过 3-7 个问题澄清需求，确认后启动执行。
</objective>

<process>
1. **检查当前状态**
   - 如果已有运行中的任务，提示用户先完成或暂停

2. **解析任务输入**
   - 如果提供文件路径 → 读取文件内容
   - 如果提供任务描述 → 直接使用
   - 如果无参数 → 询问用户要执行的任务

3. **生成澄清问题 (3-7个)**
   - 分析任务内容，识别不确定项
   - 根据任务复杂度生成 3-7 个问题
   - **逐个提问，等待用户回答后再问下一个**

```
📋 任务分析完成，请逐个回答以下问题：

[1] {根据任务内容生成的问题}

请选择或输入: _
```

用户回答后，继续下一个问题。问题类型和选项由 AI 根据任务内容灵活生成，不预设固定模板。

4. **任务拆解**
   - 根据用户选择拆解任务
   - 生成子任务列表和依赖图

5. **展示执行计划**

```
📋 执行计划

## Phase 1: 设计阶段
  └─ TASK-001: 架构设计 (15min)

## Phase 2: 开发阶段
  ├─ TASK-002: 数据模型 (20min)
  ├─ TASK-003: API 接口 (30min)
  └─ TASK-004: 前端页面 (40min)

## Phase 3: 测试阶段
  ├─ TASK-005: 单元测试 (20min)
  └─ TASK-006: 集成测试 (15min)

## Phase 4: 收尾阶段
  ├─ TASK-007: 文档编写 (15min)
  └─ TASK-008: 代码审查 (10min)

📊 统计
  总任务: 8
  预计耗时: ~2.5小时
  审批点: plan, merge
```

6. **执行模式确认**

```
⚙️ 执行模式选择

[1] 🔔 每阶段确认 (推荐)
    每个阶段完成后暂停，等待你确认
    适合重要任务，可控性高

[2] 🚀 关键节点确认
    仅在 plan/merge/deploy 时暂停
    平衡速度和控制

[3] ⚡ 全自动执行
    无需确认，自动完成所有任务
    适合简单、低风险任务

请选择执行模式 [1-3]: _
```

7. **开始执行**
   - 用户确认后，调用 CLI: `openmatrix start`
   - CLI 根据选择的模式配置审批点

</process>

<arguments>
$ARGUMENTS
</arguments>

<examples>
/om:start                      # 交互式输入任务
/om:start docs/task.md         # 从文件读取任务
/om:start "实现用户登录功能"    # 直接描述任务
</examples>

<notes>
## 问题生成规则

问题应该覆盖：
1. **目标澄清** - 明确要做什么
2. **技术选择** - 确定技术栈
3. **范围界定** - 包含/排除什么
4. **质量要求** - 测试、文档标准
5. **约束条件** - 时间、资源限制
6. **风险识别** - 潜在问题和依赖

## 执行模式对比

| 模式 | 审批点 | 适用场景 |
|------|--------|---------|
| 每阶段确认 | 每阶段结束 | 重要任务 |
| 关键节点确认 | plan/merge/deploy | 常规任务 |
| 全自动执行 | 无 | 简单任务 |
</notes>
